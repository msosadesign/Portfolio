Instant Pet Finder Signup Form

Instant Pet Finder is an idea for an app that focuses on finding cute pets for cute users. Iâ€™ve been given the task to make the registration section on a wireframe. So I used design patters to make an enjoyable and engaging flow.

This project was interesting to me because I learned about Design Patterns, how people are used to certain repeating aspects of design, and how we can help our users implementing this in our favor. I learned to apply these patterns, how to inform users about their current actions, how to organize information and overall, how to make a design less frustrating and more engaging.

Project's Problem

My objective in this project was to design the signup form for a the app. A form like this requires many design patterns to give the users an enjoyable experience and avoid overwhelming them; therefore, in theory, design patterns increase conversion rates (marketing term user for when an user completes an action). The main point is to get user to complete the registration.

Project's Audience

The user personas would be people interested in adopting a pet, who, out of the top of my mind, I guess they would be economically stable, like workers, couples and or families, looking for a new partner.

My Process

My first step was to individuate those design patterns that I can use in my favor, in particular I used: sequencing, steps left, dropdown menu, password strength meter, input feedback, categorization, progressive disclosure, input prompt, thumbnails, and recognition over recall.

I started assembling everything on the mobile version, since it's easier to reorganize it to make a desktop version rather than the other way around. My first step was the data filling, I included the necessary fields with input prompts, which indicate examples of how to format data:

On the State and City field I added a dropdown menu to keep the screen organized. On the password section I included a strenght meter to encourage users to create a strong password and there's advice on how to do it.

In the case that an user misses an input or types badly formatted text, a brief message will show up, explaining the error and how to solve it, check it out:

You can also see the progress bar on top: this helps the users know how many steps are left and encourages them to complete the registration. It also separates the tasks into more manageable steps.

On the pets section, I implemented two main patterns:

  • Recognition over recall: we show the pets in thumbnails to help the user choose one. Then, more options show up and the user can narrow its search while seeing and recognizing what he likes.
  • Progressive Disclosure: more options show up only after the users chooses, this avoids overwhelm them by showing little by little the options. The user can also continue with a simple option without narrowing it down.

Lastly, the Final Steps section helps users start doing something in the app. In this point, the user is registered and we can send him "emails about adoptable pets", which will show a feedback message when selected.

After completing the mobile version, it was easy to continue designing the desktop version. I basically rearranged everything and added images to fill the empty space. The only major difference is the pets tab, which will show an image of the pet that matches with the user's result.

Outcome and What I Learned

The final results are the previous screens, which could the be developed to create a prototype and a full design. This project taught me about creating an enjoyable flow with design patterns, using psychology principles that improve a design's experience.
